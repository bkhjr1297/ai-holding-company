## 1. Intake Workflow

**Trigger:** New client form submitted.

1. PM claims ticket within 4 business hours.
2. Verify plan + payment method.
3. Provision client SharePoint/Google Drive folder with standard structure.
4. If RSS not owned by client, create in Buzzsprout/Libsyn.
5. Send onboarding email (template) with folder links + recording guidelines.
6. Create Notion project page using Episode Tracker template.
7. SLA clock starts on first file upload, not on signup.

## 2. Episode Production Workflow

### Spark Plan (1 episode/mo)

1. PM tags `episode-uploaded` and sends editor assignment.
2. Editor light-edits within 72 hrs: remove obvious filler, sharpen first 60 seconds.
3. QA reviewer checks loudness + export.
4. PM notifies client for approval.
5. On approval, PM publishes to RSS + confirmation.

### Ignite Plan (4 episodes/mo)

1. Staggered queue: 1 episode per editing shift per month to maintain cadence.
2. Senior editor full post within 48 hrs.
3. Show notes writer generates per template within 12 hrs of edit.
4. QA + loudness check.
5. PM sends review link + show notes + chapter markers.
6. After client approval, PM schedules publish + sends confirmation.
7. Monthly analytics report runs on the last weekday of month.

### Blaze Plan (12 episodes/mo)

1. Engaged account team handles queue via shared calendar.
2. 24hr turnaround SLA from upload to review link.
3. Transcription shipped in parallel on exec order.
4. Weekly strategy sync (calendar hold) for content calendar + sponsor messaging.
5. Incident: miss 24hr SLA → client receives credit + priority review next shift.

## 3. Quality Assurance Checklist

**Pre-edit**
- [ ] Raw file format supported (WAV/MP3/AIFF/FLAC)
- [ ] All segments present (intro, body, outro)
- [ ] File naming follows convention: `{SHOW}_{EP}_raw.{ext}`

**Post-edit**
- [ ] Peak LUFS: -1 dBTP max
- [ ] Integrated LUFS: -16 ± 1 LUFS
- [ ] LRA < 12
- [ ] No clipping visible on waveforms
- [ ] Chapter markers present and accurate
- [ ] Music bed ducked under voice
- [ ] Honor bleep list (proper nouns / phrases)
- [ ] Sponsor read included (if applicable)
- [ ] Show notes spell-checked + brand term check
- [ ] Transcript vetted for names + jargon

## 4. Distribution Procedure

**One-time setup:**
- [ ] RSS feed created/handed to client
- [ ] Directory submission checklist verified
- [ ] SEO title + description agreed per style guide

**Per episode:**
- [ ] Master MP3 uploaded to hosting platform
- [ ] Show notes pasted + HTML formatted for Apple/Spotify
- [ ] Chapter markers confirmed
- [ ] Episode category/tags consistent
- [ ] Publish date/time client-confirmed
- [ ] Post-publish: test Apple + Spotify manually within 1 hr

**Directory refresh cadence:**
- Artwork: 90-day refresh cycle.
- RSS validation: run podba.se check monthly on all client feeds.

## 5. Analytics Report Procedure

1. Export from hosting platform + Spotify for Podcasters + Apple Podcasts Connect.
2. Consolidate in master spreadsheet per client.
3. Filter by: date → episode → source → completion rate.
4. Compare vs. last month baseline.
5. Write narrative framing: growth driver, friction point, experiment to run.
6. Package with sponsor-contact if applicable.

**Frequency:**
- Ignite: monthly (first business day)
- Blaze: weekly (Monday) + monthly format

## 6. Error Handling + Escalation

**File corrupt or missing:**
- Notify client immediately with upload link.
- Do not wait until SLA clock expires; `on-hold` ticket.

**Client requests out-of-scope work:**
- PM routes with change order form.
- Min rate: $125/hr editorial; 2 hrs minimum.

**Platform outage or publish failure:**
- PM notifies client + estimated backup time.
- Alternate host available (Buzzsprout staging).

**Client dispute:**
- PM compiles timeline + asset links.
- Escalates to Head of Client Success.
- Resolution SLA: 3 business days.

## 7. Onboarding New PMs

1. Shadow 2 Ignite client projects.
2. Learn directory submission flows in staging environment.
3. Pass QA exam (mock episode review).
4. Own first onboarding end-to-end under review.
5. Shadow 5 client strategy calls.

## 8. Client Communication Cadence

| Touchpoint | Spark | Ignite | Blaze |
|---|---|---|---|
| Initial onboarding email | ✅ | ✅ | ✅ |
| First episode review prompt | ✅ | ✅ | ✅ |
| Monthly analytics | ❌ | ✅ (written) | ✅ (call + written) |
| Quarterly check-in | Call | Call | Executive sync |
| Support SLA | 24hr | 12hr | 4hr priority |

**Style rules:** no overwhelm, no passive-aggressive "as per my last email," always offer decision options not open questions.

## 9. Asset Retention + Backup Policy

- Raw files: retained 12 months, then archived. Client may request earlier deletion.
- Master + show notes: retained 12 months.
- Transcripts: retained 12 months.
- All assets: cloud-synced across 2 regions, weekly integrity check.
- On client departure: export bundle delivered within 5 business days.

---

## 10. Renewal + Pause Workflow

**Renewal:**
- 30 days before contract end: PM sends renewal options email.
- Client selects plan + signs addendum.
- Updates in billing system.

**Pause:**
- 1 month per quarter max, 7-day advance notice.
- Episodes roll over or pause queue.
- Billing paused; contract terms hold.

**Cancellation:**
- 30-day written notice required.
- Final month invoiced prorata.
- 5-day export window after final invoice paid.
