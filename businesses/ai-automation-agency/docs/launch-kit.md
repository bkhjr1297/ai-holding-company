# Launch Kit Checklist — AI Automation Agency

### Pre-launch build
- [x] Product definition: SERVICE-MENU.md
- [x] ICP and qualification: ICP.md
- [x] Order-to-role blueprint: prospecting/BLUEPRINT.md
- [x] Workforce roster: agent-roster/agents.json + this file
- [x] SOP: docs/SOP.md
- [x] Pricing: pricing/plans.md
- [x] Lead sources: lead-sources/LEAD-SOURCES.md
- [x] Automation stub: automation/intake_stub.py + scripts/delivery.py
- [ ] Order form / intake site live
- [ ] Payment/configurable retainer plan live
- [ ] Email provider and calendar links confirmed
- [ ] CRM or client folder structure created for first test client
- [ ] Prompt pack and examples drafted for first 3 core workflows

### Proof of concept
- [ ] Build 1 starter automation end-to-end with a friendly SMB
- [ ] Record or document the 30-minute result
- [ ] Use that result as first case study and referral trigger

### Week 1: Pipeline fill
- [ ] 10 outbound operator conversations
- [ ] 2 partnership conversations with referral sources
- [ ] 1 short demo clip or text case study published

### Week 2–4: Validation loop
- [ ] Close first 3 paying clients
- [ ] Check delivery on-time rate; tighten architecture if below target
- [ ] If retainer conversion is low, pilot monthly reporting or optimization sprint
- [ ] Cut lead channels that do not produce qualified intent

### Month 2+: Scale
- [ ] Add paid retention and partnership channels after organic conversion proven
- [ ] Expand automation type coverage based on what clients buy most often
- [ ] Launch partnership referral program with 10–20% commission structure
- [ ] Introduce fixed-price annual package to improve cash flow and reduce churn

### No-side-effect policy
- Do not send emails, SMS, or social posts outside the explicitly agreed scope.
- Do not write to or modify client production systems without approved access.
- Do not store payment card data.
- Do not rely on scraping personal data without clear permissible purpose and disclosure.
